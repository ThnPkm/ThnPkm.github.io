前言

php反序列化造成的漏洞倒是稍微熟悉，现在看看java反序列化是如何造成漏洞的。

Java的反序列化和PHP的反序列化其实有点类似，他们都只能将一个对象中的属性按照某种特定的格式 生成一段数据流，在反序列化的时候再按照这个格式将属性拿回来，再赋值给新的对象。

php大概是序列化生成一串序列化字符串，而java是生成了字节流文件。

但Java相对PHP序列化更深入的地方在于，其提供了更加高级、灵活地方法 writeObject ，允许开发者在序列化流中插入一些自定义数据，进而在反序列化的时候能够使用 readObject 进行读取

java反序列化基础

要比较完美的实现反序列化的操作，有两个注意点

  • 如果想让一个对象能够序列化，如果想让一个对象能够序列化java.io.Serializable,这个接口是空的，相当于是一个标记作用，意味着实现了这个接口的类可以进行序列化/反序列化.
  • 设置serialVersionUID，在进行反序列化时，JVM 会把传来的字节流中的 serialVersionUID 与本地相应实体类的 serialVersionUID 进行比较，如果相同就认为是一致的，可以进行反序列化，否则就抛出序列化版本不一致的异常- InvalidCastException.

import java.io.*;

public class SerializationDemo {
    // 序列化方法
    public static void serializing(String filename) throws IOException{
        //创建对象
        Student student1 = new Student("ThnPkm",18);
        //创建文件流
        FileOutputStream fileOut = new FileOutputStream(filename);
        //创建对象输入流
        ObjectOutputStream objOut = new ObjectOutputStream(fileOut);
        //把对象序列化写入对象输入流
        objOut.writeObject(student1);
        //释放资源 关闭流
        fileOut.close();
        objOut.close();
    }

    //反序列化
    public static void unserializing(String filename) throws IOException, ClassNotFoundException {
        FileInputStream fileIn =  new FileInputStream(filename);
        ObjectInputStream objIn = new ObjectInputStream(fileIn);
        Student student2 = (Student) objIn.readObject();
        student2.show();
    }

    public static void main(String[] args) throws IOException, ClassNotFoundException {
        SerializationDemo.serializing("./data.txt");
        SerializationDemo.unserializing("./data.txt");
    }
}

class Student implements Serializable{
    public String name ;
    public int age;

    public Student(String name,int age){
        this.name=name;
        this.age=age;
    }
    public void show(){
        System.out.println(name + " is " + age + " years old.");
    }
}

反序列化利用

Java在序列化时一个对象，将会调用这个对象中的 writeObject 方法，参数类型是ObjectOutputStream ，开发者可以将任何内容写入这个stream中；

反序列化时，会调用readObject ，开发者也可以从中读取出前面写入的内容，并进行处理。

还是通过上面的demo继续 写一下

package java_seri;
import org.apache.groovy.json.internal.IO;

import java.io.*;

public class SerializationDemo {
    // 序列化方法
    public static void serializing(String filename) throws IOException{
        //创建对象
        Student student1 = new Student("ThnPkm",18);
        //创建文件流
        FileOutputStream fileOut = new FileOutputStream(filename);
        //创建对象输入流
        ObjectOutputStream objOut = new ObjectOutputStream(fileOut);
        //把对象序列化写入对象输入流
        objOut.writeObject(student1);
        //释放资源 关闭流
        fileOut.close();
        objOut.close();
    }

    //反序列化
    public static void unserializing(String filename) throws IOException, ClassNotFoundException {
        FileInputStream fileIn =  new FileInputStream(filename);
        ObjectInputStream objIn = new ObjectInputStream(fileIn);
        Student student2 = (Student) objIn.readObject();
        //student2.show();
    }

    public static void main(String[] args) throws IOException, ClassNotFoundException {
        SerializationDemo.serializing("./data.txt");
        SerializationDemo.unserializing("./data.txt");
    }
}

class Student implements Serializable{
    public String name ;
    public int age;

    public Student(String name,int age){
        this.name=name;
        this.age=age;
    }
    //重写writeObject
    private void writeObject(ObjectOutputStream outStream) throws IOException{
        outStream.defaultWriteObject();
        outStream.writeObject("I write something here.");
    }
    //重写readObject
    private void readObject(ObjectInputStream outStream) throws IOException,ClassNotFoundException{
        outStream.defaultReadObject();
        System.out.println(outStream.readObject());
    }
}
